Jak strukturovat aplikaci?

Vyber architektonický přístup pro AZKvíz

A

React SPA + Express (jeden server)

Vite/React frontend, Express + Socket.io backend — vše na jednom serveru. React build je servován přímo z Express. Jedno nasazení (Railway / Fly.io / Render).

Výhody

  • Jedno nasazení, žádné CORS
  • WebSocket funguje perfektně
  • Jednoduché na pochopení

Nevýhody

  • Build pipeline nastavit ručně
  • Frontend a backend v jednom repo
B

Next.js full-stack (monorepo)

Next.js obsluhuje frontend i API. Custom server pro WebSocket (nebo Pusher/Ably jako hosted WS). Nasazení na Vercel nebo Railway.

Výhody

  • SSR pro public view (SEO, rychlost)
  • Jeden framework, jedno repo
  • Snadné nasazení na Vercel

Nevýhody

  • Custom server komplikuje serverless nasazení
  • Více magie, těžší ladit WS
C

React SPA + Express (oddělené nasazení)

React na Netlify/Vercel (static), Express backend samostatně na Railway. Komunikují přes HTTPS + WSS.

Výhody

  • Nezávislý deploy frontendu/backendu
  • Frontend CDN = rychlé načítání

Nevýhody

  • Dvě nasazení, CORS konfigurace
  • Složitější CI/CD
Doporučení

Doporučuji možnost A — React + Express na jednom serveru. Je nejjednodušší na implementaci a nasazení, WebSocket funguje bez kompromisů a pro tento typ aplikace není SSR (Next.js) potřeba.
